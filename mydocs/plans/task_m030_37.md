# Task M030 #37 수행계획서

GitHub Issue: [#37](https://github.com/postmelee/codex-usage-analyzer/issues/37)
마일스톤: M030

## 목적

PR #44의 macOS Codex/ChatGPT 앱 번들 fallback과 GitHub `main`의 현재 public documentation을 npm registry에 함께 배포하는 `0.3.0` minor release를 준비하고 실행한다. Runtime enhancement 자체를 다시 수정하지 않고 package/version surface만 `0.3.0`으로 일치시킨다.

Release preparation PR merge 후 main commit에 `v0.3.0` tag를 만들고 기존 GitHub Actions `Publish Package` workflow와 npm Trusted Publisher OIDC를 사용해 publish한다. Registry, public `npx`, README, ECDSA signature, SLSA provenance와 GitHub Release를 순서대로 검증한 뒤 Issue #37을 종료한다.

## 배경

Issue #37 최초 계획은 GitHub/npm README 동기화를 위한 `0.2.1` patch release였다. 이후 PR #44가 merge commit `39e7e19016a0c6051aa87d6bc056689fb3d32719`로 main에 반영됐다. #37 task branch는 merge commit `5a7f6f4`에서 PR #44를 통합했고 오늘할일 충돌은 #43 완료 행과 #37 진행 행을 모두 보존해 해결했다.

PR #44는 단순 문서 변경이 아니다.

- PATH에 Codex CLI가 없을 때 macOS system/user ChatGPT·Codex 앱의 고정 후보를 탐지한다.
- `src/codex-executable.js` runtime module을 추가한다.
- App-server transport가 resolver 결과를 사용한다.
- Safe `CODEX_NOT_FOUND` 경계와 README Quick Start/Requirements를 갱신한다.
- Repository test가 43개에서 55개로 증가했다.
- npm package file이 17개에서 18개로 증가했고 resolver module이 allowlist에 추가됐다.

Issue #43은 이 변경을 기존 오류 수정이 아닌 새로운 앱 번들 fallback `enhancement`로 분류했다. Repository release guide도 backward-compatible capability addition을 minor로 정의하므로 release target을 `0.2.1`에서 `0.3.0`으로 변경하고 milestone을 M030으로 분리한다.

현재 source와 registry baseline:

- Main `package.json`, CLI/SDK version과 app-server `clientInfo.version`: `0.2.0`
- npm registry `latest`: `0.2.0`
- Existing Git tag/GitHub Release: `v0.2.0`
- Runtime/development dependency: 0개
- Main regression/package baseline: 55 tests, 18 package files
- Local runtime: Node `v24.15.0`, npm `11.12.1`
- npm `0.2.0`: Registry ECDSA signature와 SLSA provenance attestation 존재
- Source README: Codex for Open Source support, documented upstream blockquote, Codex lookup benchmark와 macOS app-only Quick Start 존재
- npm `0.2.0`: `src/codex-executable.js`와 위 README 변경이 없음
- npm `0.2.0` README exact marker:
  - `Documented upstream:`: 없음
  - OpenAI Codex app-server README URL: 없음
  - `Codex for Open Source`: 없음
  - `## Codex lookup benchmark`: 없음
  - `docs/codex-lookup-benchmark.md`: 없음

Source README와 `docs/codex-lookup-benchmark.md`의 `0.2.0`은 실제 측정 당시 package version을 기록한 dated evidence이므로 `0.3.0`으로 바꾸지 않는다.

Repository release 정책은 [`npm_release_guide.md`](../manual/npm_release_guide.md)를 진실 원천으로 사용한다. Trusted Publishing은 GitHub-hosted runner와 `id-token: write`로 short-lived OIDC credential을 사용하며 public package provenance를 자동 생성한다. Workflow에 long-lived npm token이나 `--provenance`를 추가하지 않는다.

## 리스크

- **Irreversible npm version**: npm publish 후 같은 version을 재사용할 수 없다. Strict preflight와 artifact audit 전 workflow를 실행하지 않는다.
- **External partial release**: Publish 후 검증이 실패하면 재게시하지 않고 Issue를 열린 상태로 유지해 후속 patch를 결정한다.
- **Premature mutation**: PR merge와 별도 작업지시자 승인 전 tag, workflow dispatch, publish, Release를 만들지 않는다.
- **Premature issue close**: PR에 closing keyword를 넣지 않고 registry/GitHub Release 검증 후 수동 close한다.
- **PR #44 regression**: Resolver source를 #37에서 재작성하지 않고 focused/full test와 package inclusion으로 보존 여부를 확인한다.
- **Historical evidence corruption**: Global version replacement를 하지 않고 version surface allowlist만 갱신하며 benchmark `0.2.0`을 assertion한다.
- **README false sync**: Source marker만 확인하지 않고 publish된 npm README를 registry에서 다시 읽어 exact marker를 확인한다.
- **Trusted Publisher mismatch**: Existing successful `0.2.0` provenance와 workflow boundary를 보존하고 long-lived credential을 추가하지 않는다.
- **Signature/provenance drift**: Metadata 존재와 `npm audit signatures`를 함께 확인하고 raw signature를 문서에 복사하지 않는다.
- **Dependabot conflict**: PR #40/#41이 먼저 merge되면 action version을 되돌리지 않고 최신 main을 반영한다.
- **Registry propagation delay**: Publish success 직후 `latest`/README가 지연될 수 있다. Bounded retry 후에도 불일치하면 partial-release 상태로 중단한다.

## 승인 요청 사항

- PR #44 runtime enhancement를 포함하고 release version을 `0.3.0` minor로 확정하는 방향
- Source README를 수정하지 않고 package publish로 npm public documentation을 동기화하는 방향
- Version source/test 7개 파일만 갱신하고 PR #44 resolver source와 historical benchmark `0.2.0`을 보존하는 범위
- 3개 Stage와 pre-merge/post-merge release gate 분할
- PR에 closing keyword를 넣지 않고 post-release 검증 뒤 Issue #37을 수동 close하는 절차
- PR merge 후에도 tag/publish/GitHub Release 전에 별도 작업지시자 승인을 요청하는 안전 경계

승인되면 `task_m030_37_impl.md`에서 exact version file/assertion, resolver artifact marker, preflight expectation, tag/workflow/registry/npx/signature/provenance/Release command와 커밋 메시지를 구체화한다.
