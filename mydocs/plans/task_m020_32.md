# Task M020 #32 수행계획서

GitHub Issue: [#32](https://github.com/postmelee/codex-usage-analyzer/issues/32)
마일스톤: M020

## 목적

기존 local session parser와 `UsageSnapshot v2` 중심 구조를 제거하고, 공식 Codex app-server `account/usage/read`를 호출하는 dependency-free thin CLI와 SDK로 v0.2.0을 재구성한다.

사용자는 별도 설치 없이 `npx codex-usage-analyzer`로 account usage 요약을 확인하고, `--json`으로 downstream 서비스가 바로 검증할 수 있는 versioned contract를 받을 수 있어야 한다. README는 사용자와 기여자가 제품 가치, 요구사항, 지원 지표, 보안 경계, 문제 해결 방법을 빠르게 이해하는 공개 진입점으로 다시 작성한다.

## 배경

v0.1.0은 local Codex session JSONL, asset state, skill/plugin invocation을 분석해 profile과 유사한 값을 추정한다. 이 방식은 local source coverage에 따라 profile 화면과 값이 달라지고, 실제 환경에서는 scan 시간이 길어질 수 있다.

#30에서 Codex Desktop의 비공식 profile endpoint와 credential boundary를 분석한 뒤, OpenAI가 공식화한 app-server `account/usage/read`를 현재 로그인 컨텍스트로 안전하게 호출할 수 있음을 검증했다. 공식 method는 summary 5개 범주와 nullable daily bucket을 제공하며 package가 auth file, keychain, bearer token을 직접 읽을 필요가 없다.

## 설계 방향

### Public JSON contract

CLI `--json`과 SDK는 다음 root field만 반환한다.

```json
{
  "contractVersion": 1,
  "capturedAt": "ISO_TIMESTAMP",
  "summary": {
    "lifetimeTokens": null,
    "peakDailyTokens": null,
    "longestRunningTurnSec": null,
    "currentStreakDays": null,
    "longestStreakDays": null
  },
  "dailyUsageBuckets": null
}
```

- summary 5개 field는 항상 존재하며 upstream missing/null은 `null`로 정규화한다.
- `dailyUsageBuckets: null`은 unavailable, 빈 배열은 available but empty로 구분한다.
- bucket은 `startDate`, `tokens`만 allowlist한다.
- unknown upstream field와 identity/auth 관련 field는 전달하지 않는다.
- 정수는 non-negative safe integer만 허용하고 malformed response는 safe error로 종료한다.

### App-server boundary

- `codex app-server`를 shell 없이 직접 spawn한다.
- stable API만 사용하고 `experimentalApi` capability를 켜지 않는다.
- `initialize` 응답 후 `initialized` notification과 `account/usage/read` 요청을 보낸다.
- auth file, keychain, token claim, local session directory를 직접 읽지 않는다.
- stderr와 raw RPC error message를 사용자 출력에 복사하지 않는다.
- subprocess error, early exit, timeout, RPC error, malformed JSON/shape를 stable safe error code로 정규화한다.
- 성공 또는 실패 후 stdin/readline/listener/subprocess를 정리한다.

### CLI와 SDK

- `npx codex-usage-analyzer`: 사람이 읽는 compact summary 출력
- `npx codex-usage-analyzer usage`: 동일 동작의 명시적 alias
- `npx codex-usage-analyzer --json`: 공식 downstream contract 출력
- `--help`, `--version`: app-server를 시작하지 않고 즉시 출력
- SDK: `readAccountUsage({ timeoutMs? })`
- 외부 runtime dependency를 추가하지 않는다.

### Legacy와 history

- active product surface인 `src/**`, `scripts/**`, tests, README, package metadata, CI에서 old parser와 `UsageSnapshot v2`를 제거한다.
- 과거 결정을 보존해야 하는 `mydocs/**` task 문서는 수정하지 않는다.
- npm tarball에는 새 runtime, types, 공식 `docs/`, README, license만 포함한다.

## 리스크

- **Codex version compatibility**: 오래된 Codex CLI는 method를 제공하지 않을 수 있다. safe unsupported/protocol error와 upgrade 안내를 제공한다.
- **Auth mode mismatch**: API-key-only/Bedrock은 공식 usage를 지원하지 않는다. 요구사항과 troubleshooting에 명시하고 raw upstream error를 출력하지 않는다.
- **Subprocess leak**: timeout/parse failure 시 app-server가 남을 수 있다. 모든 settle path에 cleanup을 두고 fake child test로 검증한다.
- **Contract drift**: upstream optional field 추가/누락이 downstream을 깨뜨릴 수 있다. allowlist와 fixed nullable fields, contractVersion, JSON Schema로 격리한다.
- **Breaking migration**: v0.1.0 SDK/CLI가 제거된다. v0.2.0 README와 release 문서에 breaking change를 명확히 기록한다.
- **History destruction**: 전체 저장소 zero-match를 목표로 하면 과거 조사 기록을 손상한다. active product surface만 제거하고 `mydocs` history는 보존한다.
- **README 재팽창**: release runbook이 다시 사용자 문서에 섞일 수 있다. README에는 maintainer 문서 링크와 최소 개발 명령만 둔다.
