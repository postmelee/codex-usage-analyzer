# Task M010 #14 수행계획서

GitHub Issue: [#14](https://github.com/postmelee/codex-usage-analyzer/issues/14)
마일스톤: M010

## 목적

이 task는 profile smoke 비교 결과에서 parser bug 가능성이 높은 mismatch와 Codex Desktop remote profile source와 local analyzer source 차이로 인한 mismatch를 더 명확히 구분한다.

최종 결과는 screenshot-derived baseline처럼 remote/local source가 다른 비교에서 단순 `numeric_mismatch`만 누적되지 않고, field-level result가 `source_mismatch`, `not_comparable`, `profile_parity_not_guaranteed` 같은 해석 가능한 reason을 일관되게 제공하는 것이다. `UsageSnapshot v2` schema는 변경하지 않는다.

## 배경

#6은 redacted profile baseline과 local analyzer snapshot을 비교하는 profile smoke helper를 도입했다. 현재 `src/profile-baseline.js`는 baseline에서 명시한 `not_comparable` marker와 일부 activity field의 `profile_parity_not_guaranteed` reason을 지원하지만, source가 다른 baseline에서 숫자 필드가 단순 `numeric_mismatch`로 남기 쉽다.

Codex Desktop profile은 remote account-level source를 사용할 수 있고, analyzer는 local session JSONL과 local metadata를 읽는다. 따라서 streak, top skills/plugins, thread count, activity insight 등 source-sensitive field는 비교 실패가 곧 parser bug를 뜻하지 않는다. 이번 task는 그 차이를 baseline source policy와 comparison reason taxonomy로 표현해 profile smoke 결과를 더 신뢰 가능하게 만든다.

## 범위

### 포함

- profile smoke comparison result reason taxonomy를 정리한다.
- remote profile source와 local analyzer source 차이가 큰 field에 대해 기본 reason을 보강한다.
- streak, top skills/plugins, thread count, activity insight 등 source-sensitive field의 비교 상태를 더 명확히 한다.
- baseline에서 field-level source policy를 지정하는 방법을 검토하고 필요한 최소 구현을 추가한다.
- synthetic source-mismatch baseline regression test를 추가한다.
- README의 profile parity smoke 결과 해석과 known mismatch reason을 보강한다.

### 제외

- Codex Desktop remote profile API 호출
- Codex Desktop UI 자동화 또는 screenshot OCR
- local source coverage 확장
- timezone/date bucket 산식 변경
- baseline generator 구현
- `UsageSnapshot v2` schema 변경
- 실제 사용자 raw profile 값, raw analyzer JSON, local path, 계정 식별자, 인증 정보 저장

## 설계 방향

- 기존 profile smoke helper인 `src/profile-baseline.js`와 `scripts/profile-smoke.js`를 유지하고 public SDK export는 늘리지 않는다.
- source-aware 판단은 baseline schema v1 안에서 optional metadata/policy 형태로 표현할 수 있는지 우선 검토한다. 호환성이 깨지는 schemaVersion 변경은 하지 않는다.
- baseline이 source policy를 명시하면 해당 field의 mismatch reason이 source 차이를 드러내도록 한다.
- baseline이 source policy를 명시하지 않아도 기존 snapshot diagnostics의 `profileComparison.parity: "not_guaranteed"`와 source-sensitive field 목록을 활용해 과도한 단순 mismatch를 줄인다.
- ranking field인 `skills.topSkills`, `plugins.topPlugins`는 local invocation source 부재와 remote profile aggregation 차이를 분리해 표현한다.
- README는 command 사용법보다 결과 해석과 reason taxonomy 중심으로 보강한다.
- 실제 계정 baseline은 커밋하지 않고 synthetic fixture와 test만 추가한다.

## 잠정 단계

- **Stage 1 — reason taxonomy와 baseline source policy 설계**
  - 기존 comparison status/reason 흐름을 정리한다.
  - source-sensitive field 목록과 기본 reason을 확정한다.
  - baseline field-level source policy를 optional metadata로 둘지, 기존 `not_comparable` marker 확장으로 둘지 결정한다.
  - 검증 관점: `UsageSnapshot v2` schema와 기존 baseline schema v1 호환성을 깨지 않는다.
- **Stage 2 — source-aware comparison 구현과 regression test**
  - `src/profile-baseline.js`에 source policy와 source-sensitive mismatch reason 처리를 구현한다.
  - numeric mismatch가 source 차이로 설명되는 경우와 parser bug 가능성이 높은 mismatch를 테스트로 분리한다.
  - ranking field와 absent/unavailable field의 reason을 고정한다.
  - 검증 관점: synthetic source-mismatch baseline에서 expected reason이 나온다.
- **Stage 3 — README와 fixture 문서 보강**
  - README의 result status와 known mismatch reason을 source-aware taxonomy에 맞게 업데이트한다.
  - synthetic fixture 또는 fixture README에 source policy 작성 예를 추가한다.
  - 검증 관점: 실제 profile 값, local path, account identifier, credential, screenshot 원본을 문서나 fixture에 넣지 않는다.
- **Stage 4 — 통합 검증과 최종 정리**
  - `npm test`, synthetic source-mismatch smoke, `git diff --check`를 실행한다.
  - 최종 보고서와 오늘할일 상태를 갱신한다.
  - 검증 관점: profile smoke 결과가 단순 mismatch보다 source-aware reason을 우선 제공하고 PR 준비 전 작업트리가 정리된다.

## 리스크

- **baseline schema compatibility**: source policy를 무리하게 필수화하면 기존 redacted baseline이 깨진다. optional field로 제한하고 기존 baseline도 통과해야 한다.
- **reason taxonomy 과잉 분화**: reason이 너무 많으면 smoke 결과 해석이 어려워진다. parser bug 후보와 source 차이를 구분하는 데 필요한 최소 reason만 둔다.
- **numeric mismatch masking**: source-aware reason이 실제 parser bug를 가릴 수 있다. source policy가 없는 core local-comparable field는 기존 mismatch를 유지한다.
- **privacy leakage**: 실제 screenshot-derived baseline 값을 fixture나 보고서에 복제하지 않는다. synthetic value와 safe summary만 사용한다.
- **scope creep**: local source coverage, date bucket 산식, baseline generator는 별도 task로 남기고 이번 task에서 구현하지 않는다.

## 승인 요청 사항
